# 🛍️ Customer Shopping Behavior Analysis | Python, SQL & Power BI

## 📌 Project Overview

This end-to-end data analytics project analyzes customer shopping behavior using transactional data from 3,900 purchases across multiple product categories. The objective was to uncover customer spending patterns, purchasing preferences, subscription behavior, and revenue drivers to support strategic business decision-making.

The project combines Python for data cleaning and preparation, SQL for business analysis, and Power BI for interactive dashboarding, simulating a real-world analytics workflow used in modern organizations.

---

## 🎯 Business Problem

A leading retail company wanted to better understand customer shopping behavior to improve sales performance, customer satisfaction, and long-term loyalty.

Management sought answers to key questions such as:

* Which customer segments generate the most revenue?
* How do discounts influence purchasing behavior?
* Are subscribers more valuable than non-subscribers?
* Which products and categories perform best?
* What factors drive repeat purchases and customer loyalty?

The goal was to leverage customer shopping data to identify trends, improve engagement, and optimize marketing and product strategies.

## 🐍 Data Preparation & Cleaning (Python)

The data was cleaned and transformed using Python before loading into PostgreSQL.

### Key Data Processing Steps

* Imported and explored data using Pandas.
* Performed data quality assessment.
* Handled missing values in review ratings.
* Standardized column names using snake_case convention.
* Created customer age groups through feature engineering.
* Generated purchase frequency metrics.
* Removed redundant columns after consistency checks.
* Loaded cleaned data into PostgreSQL for analysis.
